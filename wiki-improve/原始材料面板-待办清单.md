## 0. 提交前的习惯（这轮最重要的一条反馈）

**push 之前跑一次 `mvn test`（哪怕只跑改动模块的测试类）。** `8a004435` 推上来时编译不过：

1. `WikiSourceGroupBackfillRunner:61`（连同它的测试文件 4 处）还在用删掉 `scanMode` 之前的 7 参 `create()` 签名——改方法签名后漏了调用方，主代码编译失败。

这两处都是一分钟能修的机械问题，`mvn test` 会在第一秒抓住它们。修掉后 28/28 全过（含 E2E，4.3s），我们验证过。

## 1. PR-1：后端（`8d0ac970` + `8a004435` 后端部分，新分支提交）

**合并门槛：**

1. **迁移重编号**（上轮遗留，这轮已被实测证实）：跑测试时环境里恰好同时出现了 upstream 的 `V166__message_usage_detail.sql` 和你的 `V166__wiki_source_group.sql`，Flyway 当场 `Found more than one migration with version 166`、应用起不来——合并 upstream 后每台机器都会这样。先合入 upstream/dev（这次 merge 可以由倪程伟做），再把 V166/V167 重命名到空闲版本号，三套方言同步。
2. **修上面第 0 节的两个编译错。**
3. **软删 × 唯一索引的组合坑**：删除分组后，同名别名**永远建不回来**——`assertAliasAvailable` 被 `@TableLogic` 过滤看不到软删行 → 查重通过 → INSERT 撞唯一索引 → 裸 500（不是友好 400）。而"删掉分组再建同名的"是你自己场景里的常见操作。说句公道话：这个坑是评审建议（加 @TableLogic + 加唯一索引）组合出来的，不是你的失误。修法：`delete()` 里先把 alias 改成墓碑名（如 `alias + "#del#" + id`）再逻辑删，唯一性、软删约定、可重建三样全保住；补一个测试：删组 → 同名重建 → 成功。

**同 PR 小改：**

4. `updateRawGroupBatch` 归属过滤（上轮遗留）：`listByKbId` 拉全表再 `contains` 是 O(n·m)，改成 `selectList(in(id, rawIds).eq(kbId))` 一次查询；补测试：批量请求混入外库 rawId，断言实际 update 的 id 集合不含它（断言落在绑定参数上）。
5. **E2E 测试改连内存库**：`WikiRawMaterialGroupE2ETest` 现在连的是 `data/` 下共享持久 H2——每跑一次就往开发库写测试行，还依赖本地库状态。在 `@SpringBootTest` properties 里覆写 `spring.datasource.url=jdbc:h2:mem:...`。

**可讨论（不阻塞，PR 描述里提一句）**：raw 列表向 viewer 返回 `sourcePath`（服务器绝对路径）是否有意；全量扫描会重跑已迁到其它组的文件（语义写明，或 force 循环里过滤 `groupId == group.id`）；`WikiRawMaterialEntity.sourcePath` 的 "binary uploads only" 注释已过时。

> ⚠️ 重命名或改动已存在的迁移文件之后，**本地 `data/` 下的 H2 必须删掉重建**——`validate-on-migrate` 开着，改名报"已应用版本失踪"，改内容报 checksum 不匹配。这个坑这轮已经真实出现过一次。

**验收**：`mvn test` 全绿；从干净 H2 启动一次应用验证迁移 + 回填 Runner；PR 描述关联 issue #487。

## 2. PR-2：前端整体（`4705c077` + `8a004435` 前端部分，依赖 PR-1 先合）

原 PR-A 的四个小修（仍然有效）：

1. 预览加大小上限：`fileSize` 超 2MB 显示"文件过大，请下载查看"，否则大 CSV 卡死页面。
2. `batchReprocess` 循环里只调 API，结束后统一刷新一次（现在选 20 个会触发 60 次列表刷新）。
3. 文案：筛选栏"路径"→"来源类型"；"批量重扫"→"批量重新处理"（它不触发扫描）。
4. 计数口径：状态 chip"失败"只计 `failed`，与异常分区对齐。

这轮新增的三个小修：

5. **全量扫描加确认弹窗**（`mcConfirm`）：现在误点一次分组头的全量按钮 = 整组文件重新过 LLM，没有任何拦截。
6. **`weekly1` 预设表达式与文案不一致**：`CRON_PRESETS.weekly1 = '0 0 2 * * MON'`（02:00），文案写"每周一 03:00"——改其一。
7. **fileFilter 清空无效**：`form.fileFilter.trim() || null` 把空串转成 null，而 null 在后端 partial-PUT 语义下是"不改"。清空过滤器这个操作会静默失败——直接发 `form.fileFilter.trim()`（空字符串），后端非 null 即覆写。

收尾清理：

8. 删掉旧的目录输入框 + 扫描行相关的孤儿 i18n 键（`wiki.scan/scanning/scanResult/dirPlaceholder` 等）——每路径扫描已经落地，过渡入口不再需要。
